# 아키텍처 의사결정 기록 (ADR)

이 디렉토리는 TrendOps 프로젝트의 주요 기술 결정을 기록한 문서 모음입니다.

## 목적

MLOps 엔지니어 포트폴리오로서, 단순한 코드 결과물뿐 아니라 **왜 그 결정을 했는지**를 문서화합니다.
각 ADR은 다음 질문에 답합니다:

1. 무엇이 변했는가?
2. 어떤 문제를 해결하는가?
3. 왜 이 결정을 했는가?
4. 이 결정이 최선인가? (대안과 비교)

## 파일 명명 규칙

```
YYYYMMDD-<짧은-설명>.md
예: 20260407-llm-api-selection.md
    20260407-project-constitution.md
```

## ADR 작성 템플릿

새 ADR 작성 시 아래 구조를 사용합니다:

```markdown
# [제목]

**날짜**: YYYY-MM-DD
**상태**: 제안 | 승인 | 폐기

## 배경

어떤 문제 또는 상황에서 이 결정이 필요했는가?

## 결정

어떤 결정을 내렸는가?

## 대안 검토

| 대안 | 장점 | 단점 | 기각 이유 |
|------|------|------|----------|
| ... | ... | ... | ... |

## 결과 및 트레이드오프

- 이 결정으로 얻는 것
- 이 결정으로 잃는 것 / 감수하는 것
- 향후 재검토 조건 (있다면)
```
